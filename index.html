AI Agent 2026年6月26日

Vercel eve 接入 Agent Runs：可在仪表盘追踪会话、模型调用、工具调用和运行时错误

Vercel Changelog 宣布，eve 项目现在会在 Vercel dashboard 中自动出现 Agent Runs 标签。开发者可以查看每次 Agent 会话的 trigger、duration、token usage，并继续下钻到 turns、model calls、tool calls、runtime errors，以及与代码步骤相关的运行时问题。

为什么值得关注

这让新手调试 Agent 时不再只依赖最终回复或零散日志。真正要做的是先选低风险任务，完成 instrumentation，触发一次可重复运行，再用 Agent Runs 核对耗时、token、工具调用和错误来源；需要长期分析时再配置 OpenTelemetry 导出，而不是把所有生产问题都交给最终回答判断。

已扩写：Vercel eve Agent Runs 观测、调试与新手验收清单

终端 Agent 2026年6月26日

Claude Code Week 26 更新：MCP 登录命令、shell mode 回显、/rewind 与沙箱凭据提示集中增强

Claude Code Docs 的 Week 26 记录显示，v2.1.185 到 v2.1.193 期间加入 `claude mcp login/logout`，可在 shell 中认证已配置 MCP server；shell mode 会提示包含 stdout 与 stderr；`/rewind` 回退体验、模型切换、沙箱凭据警告和 `--dangerously-skip-permissions` 的可见性也有更新。

为什么值得关注

这些更新更像终端 Agent 的日常安全与可解释性修补。新手应优先验证 MCP 登录是否只影响目标 server、shell mode 是否保留足够命令输出、/rewind 是否能回到预期位置，以及沙箱凭据提示是否能阻止把敏感凭据误带进隔离环境。

候选：Claude Code Week 26 下 MCP 登录、shell mode 和 /rewind 的终端回归测试清单

AI 指标 2026年6月26日

GitHub Copilot 指标新增按 AI 采用阶段统计合并 PR 数：企业报告能看到 adoption phase 与 merge 的关系

GitHub Changelog 宣布，在此前 Copilot usage metrics API 的 AI adoption phase cohorts 基础上，enterprise 与 organization 报告新增按采用阶段统计的合并 PR 总数。新字段 `total_pull_requests_merged` 会显示某一天由对应 adoption phase 用户合并的 PR 数量。

为什么值得关注

AI 采用指标如果只看使用人数，很容易高估真实研发影响。把 adoption phase 和合并 PR 放在一起后，团队可以观察补全、Agent、multi-agent 等不同使用阶段是否真的进入交付链路；但它仍不是质量指标，必须和 review、测试失败率、回滚和人工复核结合解读。

候选：Copilot 采用阶段、合并 PR 与研发指标解读清单，结合旧指标文章后再扩写

Vercel eve Agent Runs：新手怎样给 AI Agent 留下可复查的观测记录
