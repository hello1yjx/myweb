建站工具 2026年6月15日

Vercel Functions 的 Node.js 与 Python 运行时可设置最长 30 分钟

Vercel Functions 现在支持在 Pro 和 Enterprise 团队中把 Node.js、Python 运行时的执行时间设置到 30 分钟，适合长 LLM 推理、几分钟流式响应、文档和媒体处理、OCR、网页抓取、浏览器自动化、Workflow 步骤和 Queue handler。

为什么值得关注

更长时长能减少 AI 长任务被硬超时截断的情况，但官方明确说明超过 800 秒仍处于 beta，并要求 Fluid Compute。站长应只给必要路径设置 `maxDuration`，同时验证成本、日志、幂等重试和失败提示。

选题：Vercel 30 分钟 Functions 入门，怎样给 AI 长任务设置超时、队列和验收边界

开发安全 2026年6月15日

Claude Code v2.1.178 增加参数级权限规则，并强化嵌套 Skills 与 subagent 检查

新版增加 `Tool(param:value)` 权限匹配和通配符，支持在嵌套 `.claude/skills` 中加载目录级 skill，并让最近目录的 agent、workflow、output-style 在冲突时优先生效；自动模式下 subagent 启动前也会先经过分类器检查。

为什么值得关注

这些改动直接影响 monorepo、团队模板、subagent 权限和远程控制排错。升级后不能只看版本号，应回归测试允许/拒绝规则、同名 skill、project-scope workflow 保存位置和后台会话状态。

AI 管理 2026年6月15日

Copilot 使用指标引入服务端遥测，活跃用户统计会更接近账单与活动日志

GitHub Copilot usage metrics API 现在会把服务端确认的活跃用户补进单日和 28 天企业报告，使部分过去因客户端遥测缺失而没有出现在报表里的已活跃、已计费用户被统计进 DAU。

为什么值得关注

团队可能看到顶层活跃用户数上升，但这些新增用户暂时不会带来 IDE、功能、模型或代码行等细分维度。管理者应把它看成指标口径变化，并重新解释采用率、成本归因和缺失明细。

候选：Copilot 活跃用户统计口径变化后，怎样重读 DAU、明细缺口与成本报表
