AI 终端 2026年6月23日

GitHub Copilot CLI 新终端界面 GA：Issue、PR、Gist、MCP、Skills 和插件都进入会话内管理

GitHub Changelog 宣布，Copilot CLI 在 Microsoft Build 2026 预览过的新终端界面现在正式可用：交互式会话顶部可用标签页浏览当前仓库 Issues、Pull requests 和个人 Gists；/mcp、/skills、/plugin、/settings 则把 MCP、skills、插件和设置放进终端内引导配置。官方还强调主题、窄终端适配和屏幕阅读器支持。

为什么值得关注

终端 Agent 正在从“聊天加命令”变成面向 GitHub 工作流的任务工作台。新手升级后应重点核对当前仓库、引用的 Issue/PR、启用的 MCP 工具、skills 和插件来源，先做只读总结和小补丁，再扩大到评论、审查或修复任务。

已扩写：GitHub Copilot CLI 新终端界面新手工作台验收清单

AI 模型 2026年6月23日

GitHub Copilot app 支持 BYOK：桌面 Agent 会话可连接自有模型提供方

GitHub Changelog 显示，GitHub Copilot app 现在支持 bring your own key，可在 Settings 的 Model Providers 中添加 OpenAI、Azure OpenAI、Microsoft Foundry、Anthropic、LM Studio、Ollama 或 OpenAI-compatible endpoint。添加后，自有模型会出现在模型选择器中，密钥保存在本机系统 keychain，UI 不会读回显示。

为什么值得关注

BYOK 让个人和团队可以在 Copilot app 的 Agent 会话里使用已有模型、配额、区域和数据处理边界。新手不要只把它看成“多几个模型”，还要核对本机密钥保存位置、组织策略、计费归属、是否走内部网关，以及每个会话选择了哪个模型。

候选：Copilot app BYOK 下怎样核对模型、密钥和数据边界

AI 工作流 2026年6月23日

Vercel Workflows 重做 trace viewer：可搜索、缩放并查看每一步输入输出

Vercel Changelog 宣布，Vercel Workflows 和 Workflow SDK 的 trace viewer 已重新设计，用于从头到尾检查 workflow runs：可以搜索 spans、缩放时间线、用键盘逐步查看，并点进每一步的 inputs、outputs 和 run metadata。本地开发也可通过 Workflow SDK 的 `npx workflow@beta web` 检查 runs。

为什么值得关注

AI Agent 和后台工作流的难点不只是“能跑完”，而是失败后能不能复盘每一步。对新手来说，trace viewer 的价值在于把长任务拆成可观察步骤，先看输入输出、耗时、错误和元数据，再决定是否重跑、回滚或改提示词。

候选：用 Vercel Workflows trace viewer 排查 AI 工作流失败的最小清单
