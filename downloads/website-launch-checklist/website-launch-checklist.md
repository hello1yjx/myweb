# 静态网站上线检查表

## 一、内容与链接

- [ ] 首页能在 5 秒内说明网站提供什么内容
- [ ] 导航名称清楚，所有导航链接都能打开
- [ ] 页面没有测试文案、占位文字和虚假数据
- [ ] 联系方式、版权年份和作者信息正确
- [ ] 外部链接来自可信来源，并使用 HTTPS
- [ ] 下载按钮能拿到正确文件，文件名容易理解
- [ ] 404 页面或错误入口有返回首页的方法

## 二、移动端与可访问性

- [ ] 在手机宽度下没有横向滚动
- [ ] 按钮和链接容易点击，重要按钮不依赖悬停
- [ ] 正文大小和颜色对比足够阅读
- [ ] 图片有合适的 alt 文本和宽高属性
- [ ] 键盘 Tab 可以依次访问主要操作
- [ ] 页面标题与标题层级清楚

## 三、搜索与分享

- [ ] 每个页面都有独立、准确的 title
- [ ] 每个主要页面都有 meta description
- [ ] 网站提供 favicon
- [ ] robots.txt 没有错误屏蔽正式页面
- [ ] sitemap.xml 包含需要被搜索引擎发现的页面
- [ ] 分享链接时标题和摘要符合预期

## 四、性能与安全

- [ ] 图片已压缩，不上传不必要的原始大图
- [ ] CSS 和 JavaScript 使用缓存版本或文件指纹
- [ ] 控制台没有明显报错
- [ ] 仓库中没有 API Key、密码和私人资料
- [ ] 表单、统计和第三方脚本来源可信
- [ ] 所有正式页面都通过 HTTPS 访问

## 五、发布验证

- [ ] 发布命令或部署流程已记录
- [ ] 远端仓库包含本次提交
- [ ] 部署任务状态为成功
- [ ] 使用无痕窗口打开正式网址
- [ ] 正式网站引用的是最新 CSS 和 JavaScript
- [ ] 首页、列表页、详情页和下载链接逐一验证
- [ ] 已准备回滚到上一个可用版本的方法

## 六、上线后维护

- [ ] 一周内检查一次失效链接和错误日志
- [ ] 定期更新依赖与第三方工具说明
- [ ] 删除过期、重复或没有价值的内容
- [ ] 新增内容时同步更新 sitemap 和导航
- [ ] 记录用户反馈，并优先修复影响使用的问题
