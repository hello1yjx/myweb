# VS Code 扩展选择清单

## 先问自己

- 这个扩展解决什么具体问题？
- 项目是否已经要求使用另一套工具？
- 扩展需要读取哪些文件、网络或账号权限？
- 最近是否仍在维护？
- 卸载后项目还能正常工作吗？

## 常见用途

### 格式化

- JavaScript、CSS、Markdown 项目常使用 Prettier
- Python 项目先查看项目文档使用 Ruff、Black 还是其他工具
- 团队项目优先遵循仓库已有配置

### 代码质量

- JavaScript / TypeScript：ESLint
- Python：Python 官方扩展与项目指定的检查工具
- 拼写检查：仅在确实需要时安装，并配置项目词典

### Git

- VS Code 已有基础 Git 功能
- 需要更详细历史时再考虑 GitLens 等扩展

### AI 编程

- 先确认代码是否允许发送给外部服务
- 检查模型、权限、终端命令和数据保留设置
- 涉及删除、发布、密钥和生产环境的操作必须人工复核

## 不建议

- 同时安装多个功能重复的格式化扩展
- 使用来源不明、长期不更新的扩展
- 为了“看起来专业”安装几十个不使用的扩展
- 在不了解权限时让扩展访问整个工作区
