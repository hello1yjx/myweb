# AI Agent 权限与安全检查表

## 一、任务开始前

- [ ] 任务目标、范围和完成标准明确
- [ ] 只开放完成任务需要的目录、仓库和工具
- [ ] 使用测试环境或低风险数据验证
- [ ] 密钥使用短期、最小权限凭据
- [ ] 明确禁止读取的文件和目录
- [ ] 明确禁止访问的生产系统与账号
- [ ] 删除、覆盖、发布、付款等操作必须人工确认
- [ ] 已准备日志、版本控制和回滚方式

## 二、文件系统权限

- [ ] 默认只读，确实需要时才开放写入
- [ ] 写入范围限制在指定工作区
- [ ] 不允许访问密码库、浏览器资料和私人目录
- [ ] 递归删除和批量移动必须确认绝对路径
- [ ] 不覆盖用户已有但未提交的修改

## 三、命令与代码执行

- [ ] 运行命令前能看到完整命令和工作目录
- [ ] 不执行来源不明的脚本
- [ ] 安装依赖前确认来源、版本和锁文件变化
- [ ] 不关闭防火墙、杀毒或系统安全控制
- [ ] 不使用不理解的破坏性 Git 命令
- [ ] 长时间任务有超时和停止方式

## 四、网络与外部工具

- [ ] 只连接完成任务需要的域名和服务
- [ ] 不把敏感代码或数据发送到未批准服务
- [ ] 外部 API 有速率限制、费用预算和错误处理
- [ ] 第三方连接器权限经过复核
- [ ] 自动化发布前验证目标账号和环境

## 五、运行中观察

- [ ] 代理持续说明正在做什么
- [ ] 异常、权限失败和范围变化及时停止
- [ ] 新发现的高风险步骤重新请求确认
- [ ] 不因“完成任务”而绕过限制

## 六、完成后

- [ ] 查看实际文件 diff 和命令结果
- [ ] 运行测试并检查用户真正看到的结果
- [ ] 确认未提交密钥、日志和临时文件
- [ ] 回收临时凭据和不再需要的权限
- [ ] 记录版本、验证结果和已知风险
- [ ] 出现异常时保留证据并按流程处置
