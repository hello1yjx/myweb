# 开源项目发布检查清单

## 1. README

- 一句话说明项目解决什么问题。
- 写清楚适用人群和不适用场景。
- 提供最小运行步骤，命令可以复制执行。
- 有截图、录屏或示例输出，读者能快速判断结果。
- 标明运行环境，例如 Node、Python、浏览器或系统版本。

## 2. 仓库文件

- 保留 `.gitignore`，不要提交密钥、缓存、数据库或本地日志。
- 添加许可证文件，并在 README 中说明。
- 如果项目需要配置，提供 `.env.example` 或示例配置。
- 删除无关压缩包、旧截图和临时文件。

## 3. 发布前验证

- 在干净目录重新安装依赖。
- 按 README 从头运行一次。
- 检查移动端或不同屏幕下是否还能阅读主要内容。
- 检查链接、图片、下载文件和演示入口是否可访问。
- 给 release 写明新增内容、修复内容和已知限制。

## 4. 发布后复查

- 打开线上页面或仓库首页，确认标题、说明和截图正常。
- 下载或克隆一次公开版本，确认不是只在本机可用。
- 检查 issue、讨论区或联系入口是否真实可用。
- 把验证日期、版本和结果写进维护记录。
