# AI 工具来源核验表

这个资料包用于在试用新的 AI 编程、建站或效率工具前，先确认来源、权限、费用、数据处理和验收方式。它适合个人学习、课堂演示和团队内部选型讨论。

## 文件清单

- `source-checklist.md`：逐项核对工具来源、官方文档、更新记录、权限和数据边界。
- `tool-evaluation.csv`：记录候选工具的用途、来源、风险和试用结论。
- `source-record-template.md`：保存一次工具核验记录，方便以后复查。

## 使用步骤

1. 先打开工具的官方网站、官方文档或官方仓库，不从无来源转载页面开始判断。
2. 用 `source-checklist.md` 核对安装入口、账号权限、文件访问范围、联网能力和费用说明。
3. 把试用结论写进 `tool-evaluation.csv`，不要只记录“好用”或“不好用”。
4. 用 `source-record-template.md` 留下来源链接、测试环境和人工验收结果。

## 适用边界

这个包不能替代法律、安全或采购审查。它的作用是帮助新手把“看起来很强的工具”拆成可验证的信息，再决定是否继续深入使用。
