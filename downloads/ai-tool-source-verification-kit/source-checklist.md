# AI 工具来源核验清单

## 1. 来源是否清楚

- 官方网站、官方文档或官方仓库可以打开。
- 发布主体和产品名称一致，没有被二手页面改名包装。
- 安装入口来自官方说明，不来自不明网盘或压缩包。
- changelog、release notes 或文档更新时间可以找到。

## 2. 权限是否可控

- 明确工具会读取哪些文件、目录、浏览器页面或云端数据。
- 明确工具是否可以运行命令、修改文件、提交代码或连接外部服务。
- 高风险操作前有确认步骤，或者可以关闭自动执行。
- 可以查看运行日志、工具调用记录或错误信息。

## 3. 数据和账号边界

- 不把私人密钥、账号密码、客户数据直接交给工具测试。
- 确认是否会把代码、日志、提示词或文件上传到云端。
- 了解免费额度、付费方式、团队权限和数据保留说明。
- 试用账号和正式账号分开，先用低风险项目验证。

## 4. 新手验收

- 让工具完成一个只读任务，例如解释项目结构或整理文档。
- 再让工具完成一个小范围可回滚任务，例如修改一段示例代码。
- 检查输出是否保留来源、步骤、限制和人工确认点。
- 记录失败场景：联网失败、长响应超时、权限拒绝、文件冲突。

## 5. 结论

- 可以继续使用：说明适合的任务和不适合的任务。
- 暂缓使用：说明还缺少哪些证据或权限控制。
- 不建议使用：说明具体风险，不只写“感觉不安全”。
