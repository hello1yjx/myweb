# AI 任务提示词模板

使用时请删除不适用项，并补充真实材料。不要把密码、密钥或私人敏感信息直接交给 AI。

## 1. 学习解释模板

```text
请用适合【我的基础水平】的方式解释【主题】。

我的已知基础：
- （填写）

我真正想解决的问题：
- （填写）

输出要求：
- 先用一句话说明核心概念
- 再给一个最小示例
- 指出 3 个常见误区
- 最后给我一个可以自己完成的小练习

如果某个结论可能已经变化，请明确说明并建议核对官方来源。
```

## 2. 代码问题诊断模板

```text
请诊断下面的问题，并优先找到根因，不要直接重写整个项目。

目标行为：
- （填写）

实际行为：
- （填写）

错误信息：
- （填写）

相关代码和文件：
- （填写）

环境：
- 操作系统：
- 语言或框架版本：
- 复现步骤：

请按以下顺序回答：
1. 最可能的根因
2. 如何验证判断
3. 最小修改方案
4. 修改后的验证步骤
5. 仍然存在的风险
```

## 3. 小功能实现模板

```text
请在现有项目中实现【功能】。

用户目标：
- （填写）

必须支持：
- （填写）

不需要支持：
- （填写）

现有约束：
- 遵循项目已有结构和样式
- 不修改无关文件
- 不引入不必要依赖

完成标准：
- （填写）

请先阅读相关文件，再实施、测试并说明改动。
涉及删除、覆盖、发布或生产数据时，必须先提醒我确认。
```

## 4. 内容写作模板

```text
请为【目标读者】写一篇关于【主题】的内容。

读者当前问题：
- （填写）

希望读者看完能做到：
- （填写）

语气：
- 清楚、克制、适合新手
- 不使用无法验证的夸张结论

结构：
- 开头说明为什么值得关注
- 分步骤解释
- 给出实际例子
- 写清限制和风险
- 保留官方来源链接

禁止：
- 编造数据
- 伪造引用
- 把预测写成已发生事实
```

## 5. 资料整理模板

```text
请把下面材料整理成可执行的资源清单。

材料：
- （填写）

筛选规则：
- 优先官方和原始来源
- 删除重复项
- 标记是否免费、是否需要登录、最后更新时间
- 对不确定内容明确写“待确认”

输出字段：
- 名称
- 适合谁
- 能解决什么问题
- 官方链接
- 使用限制
- 推荐下一步
```

## 6. 方案评审模板

```text
请评审下面方案，不要默认同意。

方案目标：
- （填写）

当前设计：
- （填写）

已知限制：
- （填写）

请重点检查：
- 是否真的解决用户问题
- 是否存在更简单的方案
- 权限、安全和隐私风险
- 维护成本
- 测试与回滚是否充分

输出：
1. 按严重程度排序的问题
2. 推荐修改
3. 可以保留的部分
4. 最终验收清单
```
