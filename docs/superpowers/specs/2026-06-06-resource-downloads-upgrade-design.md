# 资源下载页升级设计

## 目标

把资源页从“官方链接导航”升级为真正可下载、可复用、可持续维护的资源中心。

## 页面结构

1. 首屏说明资源来源、使用方式和更新时间。
2. “本站原创下载包”展示 6 个 ZIP，按钮直接下载。
3. “官方精选资料”保留少量长期可靠的一手来源，按钮明确标记为外部链接。
4. “精选学习路线”只展示 6 个适合继续阅读的站内专题，避免重复铺满全部文章。

## 下载包

- 静态网站上线检查表
- Git 与 GitHub 中文速查包
- VS Code 新手配置包
- AI 任务提示词模板包
- AI Agent 权限与安全检查包
- 个人静态网站起步模板

每个下载包都包含 README 和实际可编辑文件，并在 `downloads/` 下同时保留源文件目录与 ZIP。

## 数据与交互

- 在 `siteData` 新增 `downloads` 数据数组，和外部 `resources` 分开管理。
- 下载卡片展示格式、包含文件、适用场景、更新时间和直接下载按钮。
- 官方资料卡片继续使用外链，并明确“外部官方”。
- 首页资源推荐优先展示原创下载包。
- 下载链接使用原生 `<a download>`，不伪造下载次数，不依赖后端。

## 视觉与可访问性

- 延续现有白底、蓝色强调、圆角卡片和 SVG 图标语言。
- 下载卡片使用更明显的文件信息和主操作按钮。
- 移动端单列，按钮保持足够触控面积。
- 外部链接与直接下载使用不同文案和图标提示。

## 验证

- JavaScript 语法检查。
- ZIP 存在且可解压检查。
- 页面数据顺序、下载路径和站内文章 ID 检查。
- `resources.html` 动态渲染检查。
- GitHub Pages 上线后逐一请求 6 个 ZIP 和资源页。
